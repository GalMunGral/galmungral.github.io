2D Vector Graphics: A Survey

Wenqi He · Research Software Engineer, NCSA, University of Illinois

Extracted from my presentation slides by Claude (Anthropic).

A vector graphics renderer takes a list of shapes — each defined by curves and a fill rule — and produces a pixel buffer. The problem looks deceptively simple. In practice it spans a surprisingly wide slice of computer science: computational geometry, numerical algorithms, GPU architecture, and signal processing. This is an algorithmic tour from first principles to the state of the art.

Part I — The Inside-Outside Test

The fundamental question in vector rasterization is: given a closed curve and a point, is the point inside or outside? The standard answer is the winding number — the number of times the curve wraps counterclockwise around the point. Formally, for a closed curve γ and a point p,

$$w(\gamma, p) = \frac{1}{2\pi} \oint d\theta$$

Two fill rules are in common use. The non-zero rule considers a point inside if the winding number is non-zero; the even-odd rule considers it inside if the winding number is odd. They agree whenever the curve has no self-intersections.

To implement this for curves defined by cubic polynomial segments, we count signed crossings of a ray cast from the point. Each crossing contributes +1 or −1 depending on the direction of travel. Finding the crossings requires solving for the roots of a cubic polynomial — the t values where the curve crosses the ray — which can be done analytically or by a robust numerical method.

Part II — Software Rasterization

In practice, the curves are Bézier curves. A cubic Bézier is defined by four control points p0,…,p3 and parameterized using the Bernstein basis:

$$B(t) = (1-t)^3 p_0 + 3t(1-t)^2 p_1 + 3t^2(1-t)\, p_2 + t^3 p_3$$

The most elegant algorithm for evaluating this is de Casteljau’s algorithm, which replaces the polynomial with repeated linear interpolation: 

def bezier(t, p0, p1, p2, p3):
    p4 = lerp(p0, p1, t)
    p5 = lerp(p1, p2, t)
    p6 = lerp(p2, p3, t)
    p7 = lerp(p4, p5, t)
    p8 = lerp(p5, p6, t)
    return lerp(p7, p8, t)

def lerp(a, b, t):
    return (1 - t) * a + t * b

Beyond evaluation, de Casteljau’s algorithm also gives a clean subdivision: the intermediate points at parameter t are the control points of the two sub-curves on [0,t] and [t,1]. This underlies adaptive subdivision, which recursively splits a curve until each piece is flat enough to be approximated by a line segment: 

def flatten(curve):
    if flatness(curve) < eps:
        vertices.append(curve.p3)
    else:
        l, r = subdivide(curve)
        flatten(l)
        flatten(r)

Once the curves are flattened to polygons, scanline rasterization fills them. For each scanline y, we find all edge crossings, sort them by x, and scan left to right, tracking the winding number. Pixels between crossings where the winding number satisfies the fill rule are filled: 

def rasterize(polygon, buf):
    for y in polygon.y_range:
        xs = polygon.intersect(y)
        wn = 0
        for i in range(len(xs) - 1):
            wn += xs[i].sign
            if not inside(wn): continue
            for x in range(xs[i].x, xs[i+1].x):
                buf[y, x] = over(polygon.color(x, y), buf[y, x])

The naive approach recomputes all intersections for each scanline. The active edge list optimization maintains only the edges currently crossing the scanline and updates them incrementally as y advances, reducing the work per scanline from O(n) to O(k) where k is the number of active edges.

Part III — Hardware Acceleration

Modern rendering engines — Chrome’s RenderingNG, Mozilla’s WebRender — off-load rasterization to the GPU. The GPU pipeline runs a vertex shader once per vertex (in parallel) and a fragment shader once per pixel (in parallel). Two broad strategies have emerged for vector graphics on this pipeline.

Triangulation. Approximate each shape with triangles. The fragment shader is trivial — every fragment inside a triangle receives the fill color. The difficulty lies in the tessellation itself, particularly for curved boundaries. Curves are approximated to a fixed tolerance, which breaks down at high zoom levels.

Implicitization. Instead of approximating, encode the curve as an implicit function and evaluate it per fragment. For a circle of radius r: 

def fragment(x, y):
    if x*x + y*y - r*r <= 0:
        return color

Loop and Blinn (2005) generalized this to arbitrary cubic curves using a GPU-friendly implicit formulation, enabling resolution-independent rendering without tessellation.

A third approach, pioneered by Nehab and Hoppe (2008) and extended by Ganacim et al. (2014), works entirely in compute kernels. Each pixel independently computes its winding number by iterating over all relevant curve segments and accumulating signed crossings. Acceleration structures — regular spatial grids or shortcut trees — prune the set of candidate curves per pixel. An additional tiling pass precomputes the winding parity of each tile from its boundary alone, allowing most pixels within a tile to skip the curve iteration entirely.

NVIDIA’s NV_path_rendering (NVPR) exposes path rendering as a first-class GPU API, using stencil-buffer operations to accumulate winding numbers and cover passes to fill resolved pixels.

Part IV — Signal Processing and Further Connections

Rasterization is sampling. A rasterizer converts a continuous 2D signal (the filled shape) to discrete pixel values at a finite sampling frequency. The Nyquist–Shannon theorem applies: if the spatial frequency of the shape boundary exceeds half the pixel density, aliasing results. The standard remedy is bandlimiting — blurring the signal before sampling — which in the rasterization context means computing anti-aliased coverage rather than a binary inside/outside value.

A more recent connection is to differentiable rendering. Li et al. (2020) show how to propagate gradients through a rasterizer, enabling vector graphics to be optimized by gradient descent. The key challenge is that the rasterization function is discontinuous at shape boundaries; automatic differentiation handles the smooth interior but requires special treatment at the boundary.
